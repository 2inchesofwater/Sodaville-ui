How to use inputs

Text-entry inputs

Lets users enter and edit text.

HTML provides a variety of input types to suit a range of formats, such as email addresses, telephone numbers, and long-text. The user's input can be validated automatically by the browser. Use the right input type wherever possible

Labels

In HTML, add a label with a 'for' property to all inputs (which must have a matching 'id' property). This is important for accessibility

In Sodaville UI, the label always sits above its input field. We do not follow Material Design's "floating labels" pattern

A label above (or before) an input does not need a special class name.

A label below (or after) an input requires a class name 'help' or 'error'

Field

Every input unit - for example: a label, input, and a help label - must be wrapped inside a div element with the class name 'field'

Fields

Sometimes it's helpful to have several input fields side-by-side. For instance, when asking a user for their first and last name.

Wrap the group of input units inside a div element with the class name 'fieldsCollection', plus a unique class name to describe the function of the inputs

The behaviour of input fields across a variety of device sizes tends to be quite specific. Therefore, writing a feature-specific set of grid styles will be necessary.
