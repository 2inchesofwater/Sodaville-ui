How to use buttons

Dominant

A Dominant button is only used when the user is about to commit to a permanent change

We pair Dominant buttons with Naked buttons, to give even greater emphasis to the primary call-to-action

Never pair a Dominant with a Base or Recessive button

Base

Use a Base (or default) button when a button appears on its own (but always within a Buttons Collection)

We pair Base buttons with Naked buttons when the UI requires a Cancel or Back option

We pair Base buttons with Recessive buttons when it's unclear what the user's intention will be
